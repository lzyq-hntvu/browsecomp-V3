## 二、阿里巴巴的**三大核心技术创新**（可直接借鉴）

### 1. **AgentFounder：全自动问题生成引擎**

这是比BrowseComp原始方法更先进的核心系统：

- **实体锚定的开放世界记忆（Entity-Anchored Open-World Memory）**
  - 将非结构化文本（网页、文档）重组为以实体为索引的知识陈述
  - 从实体簇中采样生成问题，覆盖事实检索、数值计算、多跳推理等类型
  
- **零API成本生成**：全程不调用外部搜索API，离线完成问题合成

### 2. **FAS（一阶动作合成）：低成本高质量数据生成**

针对胡云舒小组的低成本需求，这是最实用的技术：

```python
# FAS的核心流程（基于技术报告整理）
1. 知识→问题转换：从知识图谱采样实体簇生成多样化问题
2. 规划数据合成：
   - 为每个问题生成K个不同的分析思路和首步动作（工具调用）
   - 只生成"思考链+工具调用意图"，**不实际执行API调用**
   - 使用LLM-as-Judge进行拒绝采样过滤
3. 推理数据合成：
   - 将问题分解为子问题，生成推测性答案A1
   - 结合必要知识修正逻辑，生成最终答案A2
```

**成本优势**：相比OpenAI方法需要实际调用模型验证每个问题，FAS通过**离线合成+拒绝采样**，成本降低**90-95%**。

### 3. **HAS（高阶动作合成）：提升问题复杂度**

解决BrowseComp风格问题需要的高难度推理：

- 将单一路径轨迹重建为**多步决策数据**
- 在每个步骤生成N个可选的下一步动作，形成(N+1)×K个推理选项
- 教会模型灵活决策而非简单模仿，增强探索能力

## 三、具体实施建议（针对胡云舒小组）

基于阿里巴巴开源的方法论，我建议你们采用以下**混合策略**：

### 阶段1：知识图谱构建（1-2周）
- 使用**实体锚定方法**处理学术文献/网页数据
- 构建"实体→陈述"映射，而非复杂的关系抽取
- **工具推荐**：NetworkX + 轻量级LLM进行实体链接

### 阶段2：自动化问题生成（核心）
采用**类FAS方法**：
- **离线生成**：用开源模型（如Qwen2.5-7B）批量生成问题候选
- **难度控制**：借鉴阿里巴巴的"基于集合论的形式化方法"，通过约束组合控制复杂度
  - 简单：2跳路径（作者→论文）
  - 中等：3跳+时间约束（作者→机构→论文+年份）
  - 困难：4跳+数值约束（引用次数区间+多条件组合）
- **拒绝采样**：用LLM作为评判器自动过滤低质量问题

### 阶段3：质量验证（低成本版）
替代OpenAI昂贵的三重验证：
- **自动搜索验证**：用轻量级搜索引擎验证答案可检索性（而非可解决性）
- **一致性检查**：多次采样验证答案稳定性
- **人工抽样**：仅对10%的高难度问题进行人工审核
